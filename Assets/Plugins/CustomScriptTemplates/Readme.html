OVERVIEW

After importing the package, you will have an additional MenuItem "Custom Script Templates" under the "Create" menu.

(Right click into the "Project" window.)

The position of the "Custom Script Templates" menu item might be different from the one on the right, depending on the Unity version.

This MenuItem holds all Templates.

You can customize the default ones, delete items you don't want, or create your own Templates.

TEMPLATES

Templates are stored in .txt-files under: Assets/Plugins/CustomScriptTemplates/Templates/Resources

#SCRIPTNAME# will be replaces with the actual name you gave the file.

#NAMESPACE# will be replaced with the Namespace.

(#NAMESPACE# is optional, you can remove it if you don't want a Namespace.)

The Namespace will be set based on the folder structure where the file is located.

The root Namespace from the Unity project or any Assembly Definition file will be added as well.

(Only if a root Namespace is set and the file is located under a Assembly Definition file.)

If you're working in Rider or have ReSharper installed,
folder that are excluded as a Namespace provider will not be included in the Namespace.

SETTINGS

The "Settings" asset will be under: Assets/Plugins/CustomScriptTemplates

The "Templates" list contains all .txt-file Templates.

After you created a new Template,
you have to add it to the "Templates" list and press the "Compile Menu Items" button.

If you want to be able to create Text/Json/XML files from the drop-down menu, just check the box in the inspector.

The "Indent Amount" is the number of whitespaces between the border and the class declaration.

(Will only have an effect if you're using a Namespace.)
